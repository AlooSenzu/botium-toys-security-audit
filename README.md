# 🔐 Botium Toys Security Audit

## 📌 Overview
This project presents an internal security audit conducted for a fictional company, Botium Toys, using the NIST Cybersecurity Framework (CSF).

The objective was to identify risks, assess existing controls, and evaluate compliance with industry standards such as PCI DSS, GDPR, and SOC.

---

## 🎯 Objectives
- Assess current assets and risks
- Evaluate existing security controls
- Identify gaps in compliance
- Recommend improvements to strengthen security posture

---

## 🧩 Key Findings

- ❌ No encryption for sensitive customer data (credit card info)
- ❌ No intrusion detection system (IDS)
- ❌ No disaster recovery plan or backups
- ❌ Weak password policies and no centralized password management
- ❌ Lack of least privilege and separation of duties

- ✅ Firewall and antivirus are implemented
- ✅ Physical security controls (locks, CCTV, fire systems) are in place

---

## ⚠️ Risk Summary
The organization has a **high risk score (8/10)** due to:
- Poor access control
- Lack of encryption
- Missing critical controls
- Compliance gaps

---

## 🛠️ Recommendations

- Implement encryption for sensitive data
- Enforce least privilege and separation of duties
- Deploy an Intrusion Detection System (IDS)
- Introduce backups and disaster recovery planning
- Strengthen password policies and implement password management
