# 🔐 Security Risk Assessment: Network Hardening After Data Breach

## 📌 Overview
This project analyzes a security breach at a social media organization where sensitive customer data was compromised due to multiple security weaknesses. The objective is to identify key vulnerabilities and recommend effective network hardening strategies to prevent future attacks.

---

## 🎯 Objectives
- Identify critical security vulnerabilities
- Apply network hardening techniques
- Recommend preventative security controls
- Demonstrate risk assessment and mitigation skills

---

## 🛠️ Tools & Concepts
- Network hardening practices
- Access control and authentication
- Firewall configuration
- Security policy enforcement

---

## 🔍 Identified Vulnerabilities

The following vulnerabilities were discovered:

1. Employees share passwords  
2. Admin database password is set to default  
3. Firewalls lack proper traffic filtering rules  
4. Multifactor authentication (MFA) is not implemented  

---

## 🚨 Risk Analysis

These vulnerabilities expose the organization to:

- Unauthorized access to sensitive data  
- Increased likelihood of brute force attacks  
- Network exploitation due to lack of filtering  
- Weak authentication security  

---

## 🛠️ Recommended Hardening Measures

### 1. Enforce Strong Password Policies
**Description:**
- Require complex passwords
- Prohibit password sharing
- Enforce password rotation policies  

**Effectiveness:**
- Reduces risk of credential compromise  
- Prevents unauthorized access  
- Strengthens authentication security  

**Implementation Frequency:**
- Enforced continuously  
- Password updates every 60–90 days  

---

### 2. Implement Multifactor Authentication (MFA)
**Description:**
- Require users to verify identity using multiple factors (e.g., password + OTP)

**Effectiveness:**
- Adds an additional security layer  
- Prevents account compromise even if passwords are stolen  

**Implementation Frequency:**
- Continuous enforcement across all systems  

---

### 3. Configure Firewall Rules (Traffic Filtering)
**Description:**
- Define rules to allow/deny traffic based on IP, ports, and protocols  
- Block unauthorized inbound and outbound traffic  

**Effectiveness:**
- Reduces attack surface  
- Prevents malicious traffic from entering the network  
- Improves network monitoring and control  

**Implementation Frequency:**
- Continuous monitoring  
- Rule reviews monthly or after security events  

---

## 📊 Conclusion

The organization’s breach was primarily caused by weak authentication practices and lack of network controls. By implementing strong password policies, enforcing MFA, and configuring firewall rules, the organization can significantly reduce its risk exposure.

These measures demonstrate how proactive network hardening can prevent unauthorized access, mitigate attacks, and improve overall security posture.
